# JavaScript Interactive Site Rubric

## Score Breakdown

| **Criterion** | **Excellent (4)** | **Proficient (3)** | **Developing (2)** | **Beginning (1)** | **Missing (0)** |
|---|---|---|---|---|---|
| **Two-Page Structure & Navigation** | Both pages are created, fully linked to each other, and navigation between them is intuitive and error-free. | Both pages exist and are linked; navigation works but may be unclear or inconsistent in placement. | Pages exist but linking is incomplete or navigation is broken on one page. | Only one page is present, or pages exist but cannot navigate between them. | No pages submitted or no attempt made. |
| **CSS Grid Layout & Responsiveness** | CSS Grid is properly implemented on both pages; layout is fully responsive and adjusts elegantly on mobile, tablet, and desktop screens. | CSS Grid is used on both pages; layout is mostly responsive with minor issues at certain breakpoints. | CSS Grid is used on one or both pages but responsiveness is limited or inconsistent; some layouts break at smaller screen sizes. | CSS Grid is attempted but poorly implemented; layout is not responsive or appears broken. | No CSS Grid used; layout relies on older methods (floats, tables, or no structure). |
| **JavaScript Functions for Dynamic Styling** | Multiple JS functions dynamically change page styling (colors, fonts, visibility, etc.) in response to user interaction; code is clean and well-organized. | JS functions are used to dynamically style content; most interactions work as intended with minor bugs or limited variety. | JS functions are present but only one or two interactions work; some styling changes may not be dynamic or responsive to user actions. | JS functions are attempted but mostly non-functional; styling changes are minimal or do not respond properly to user input. | No JavaScript used or no attempts at dynamic styling. |
| **Animation or Slideshow** | A smooth, well-designed slideshow or animation enhances the site's topic; transitions are fluid, timing is appropriate, and it engages the user. | A functional slideshow or animation is present; it works on one page and relates to the topic, though transitions may be basic. | A slideshow or animation is present but may be choppy, timing may be off, or it lacks clear connection to the site's topic. | An animation or slideshow is attempted but is incomplete, broken, or barely functional. | No animation or slideshow is present. |
| **Code Quality & Organization** | Code is clean, well-commented, properly indented, and easy to read; variable and function names are meaningful and follow conventions. | Code is mostly organized and readable; comments are present for complex sections; minor inconsistencies in naming or formatting. | Code works but lacks comments or clear organization; some variable names are unclear; inconsistent formatting throughout. | Code is disorganized, difficult to follow, and lacks comments; naming conventions are unclear. | No code submitted or code is incomprehensible. |
| **Testing & Debugging** | All features have been tested across browsers/devices; no console errors; all interactive elements respond correctly and reliably. | Most features work correctly; minor console errors that don't affect functionality; interactions are generally reliable. | Some features work; noticeable bugs or console errors that don't completely break the site; inconsistent behavior. | Multiple bugs or errors prevent features from working properly; console errors are present; little evidence of testing. | No testing evident; site is non-functional or severely broken. |
| **GitHub Repository & Documentation** | Repository is well-organized with clear folder structure; all files (HTML, CSS, JS, images) are present and properly named; commit history shows regular, meaningful updates. | Repository contains all necessary files and is organized; commit history shows multiple uploads, though messages may lack detail. | Repository exists with most files present; folder structure is unclear; few commits or minimal commit messages. | Repository exists but is disorganized or missing files; very few commits or no meaningful version history. | No GitHub repo submitted or repo is empty. |
| **Classroom-Friendly Topic & Content** | Topic is appropriate, engaging, and well-developed; content is accurate and relevant to both pages; site reflects effort and creativity. | Topic is appropriate and content is present on both pages; mostly accurate with adequate depth. | Topic is acceptable but content is minimal or one-page feels underdeveloped; some inaccuracies present. | Topic is vague or content is sparse; minimal effort evident in development. | No topic chosen or inappropriate content submitted. |

---

## Scoring Guide

**Total Points: 32**

- **28–32 points (A):** Excellent — All requirements met with high quality; site is polished, interactive, and demonstrates strong understanding of HTML, CSS, and JavaScript.
- **24–27 points (B):** Proficient — All major requirements met; minor issues with responsiveness, interactivity, or code organization.
- **20–23 points (C):** Developing — Most requirements attempted; noticeable gaps in functionality, responsiveness, or code quality; site is usable but has bugs.
- **16–19 points (D):** Beginning — Significant gaps in requirements; multiple broken features or incomplete implementations; site barely functions.
- **Below 16 points (F):** Does not meet expectations — Missing major components or requirements; site does not function as intended.

---

## Key Reminders for Self-Assessment

Before submitting, ask yourself:

- **Functionality:** Do all links work? Do all JavaScript interactions respond as expected? Does the slideshow/animation run smoothly?
- **Responsiveness:** Does my site look good on my phone, tablet, and laptop? Are there any awkward gaps or overlapping elements?
- **Code Quality:** Would someone else be able to understand my code? Are my variable names clear? Did I add comments for complex logic?
- **Completeness:** Are all files uploaded to GitHub? Did I include image files? Is my commit history visible?
- **Topic Appropriateness:** Is my topic suitable for a classroom setting? Is the content accurate and meaningful?
